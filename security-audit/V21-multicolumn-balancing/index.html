V21 · Superlinear multicolumn balancing in content size

Medium   Denial of service (CPU)

Status open
Key file source/layout/multicolumnbox.cpp

Nature

Column balancing is superlinear in content size, even at a legitimate column-count: large multicolumn content takes tens of seconds. Distinct from V12 (which bounds the number of columns): here it's the cost of balancing itself.

Location

  • source/layout/multicolumnbox.cpp — balancing algorithm (repeated relayout)

Risk & impact

Impact: CPU hang/slowness (DoS) from large multicolumn content.

Reproduction

  1. Generate poc/make-multicol.py (large content, columns:3) → render → slow pre-fix, bounded post-fix.

Example files:

  • poc/make-multicol.py

Proposed fix

Bound the number of balancing iterations (or improve the algorithm), configurable, without breaking balancing for normal documents.

Configuration (default + configurable)

Bounded number of balancing iterations (sane default, configurable).

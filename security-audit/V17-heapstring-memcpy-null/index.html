V17 · memcpy on a null pointer for an empty string

Low   UB / robustness

Status open
Key file source/heapstring.h:72

Nature

Heap::createString passes string_view::data() to memcpy. For a default-constructed empty std::string_view, data() is nullptr, and memcpy(dst, nullptr, 0) is undefined behavior (even with a zero size). Reachable via a CSS content escape that produces an empty string at the end of input (e.g. content:"\ followed by EOF).

Location

  • source/heapstring.h:72 — Heap::createString: memcpy(dst, sv.data(), sv.size()) with a null data()

Risk & impact

Impact: UB (flagged by UBSan), benign in practice. Low severity.

Reproduction

  1. Generate poc/make-poc.py (CSS content ending with a backslash right before EOF), render under UBSan -> runtime error: null pointer passed to memcpy.

Example files:

  • poc/make-poc.py

Proposed fix

Only call memcpy if size > 0 (and/or data is non-null).

Configuration (default + configurable)

Robustness fix (no knob).

V17 · memcpy sur pointeur null pour une chaine vide

Basse   UB / robustesse

Statut corrige
Fichier cle source/heapstring.h:72

Nature

Heap::createString transmet string_view::data() a memcpy. Pour un std::string_view vide construit par defaut, data() vaut nullptr et memcpy(dst, nullptr, 0) est un comportement indefini (meme avec une taille nulle). Atteignable via un echappement CSS content produisant une chaine vide en fin d'entree (ex. content:"\ suivi de EOF).

Localisation

  • source/heapstring.h:72 — Heap::createString: memcpy(dst, sv.data(), sv.size()) avec data() null

Risque & impact

Impact : UB (signale par UBSan), en pratique benin. Faible severite.

Reproduction

  1. Generer poc/make-poc.py (CSS content finissant par un backslash juste avant EOF), rendre sous UBSan -> runtime error: null pointer passed to memcpy.

Fichiers d'exemple :

  • poc/make-poc.py

Correctif propose

Ne appeler memcpy que si size > 0 (et/ou data non null).

Configuration (defaut + configurable)

Correctif de robustesse (pas de knob).

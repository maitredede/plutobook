V16 · XML billion-laughs protection depends on the expat version

Info   Info / dependency

Status fixed
Key file meson.build:8-11

Nature

Good news: no XML_SetExternalEntityRefHandler or XML_SetParamEntityParsing → XXE is unreachable. But meson.build prefers the system expat; billion-laughs protection (internal entity expansion) therefore depends on the version (protected by default from expat 2.4.0 onward; the bundled fallback 2.7.3 is).

Location

  • meson.build:8-11 — dependency('expat', fallback: ...) prefers the system expat
  • source/xmlparser.cpp:45-64 — no external entity handler (XXE unreachable)

Risk & impact

Impact: memory DoS via entity expansion if the host provides an expat < 2.4.0.

Reproduction

  1. Render poc/lol.xml with an expat < 2.4.0.

Example files:

  • poc/lol.xml

Proposed fix

meson.build now pins dependency('expat', version: '≥2.4.0', ...): a system expat that is too old is rejected by meson, which then falls back to the bundled subproject (already protected). As a belt-and-suspenders complement, XMLParser::parse (source/xmlparser.cpp) now explicitly sets, right after XML_ParserCreateNS: XML_SetBillionLaughsAttackProtectionMaximumAmplification(parser, 100.0f) and XML_SetBillionLaughsAttackProtectionActivationThreshold(parser, 8*1024*1024) (the same values as expat ≥ 2.4.0's own defaults, set explicitly rather than relying solely on the library's defaults). Both calls are guarded by #if defined(XML_MAJOR_VERSION) && (… >= 2.4.0) (checked after including expat.h) so the code keeps compiling if the version floor is ever lowered. expat.h only declares these two prototypes if XML_DTD or XML_GE is defined by the includer (matching how the linked expat was built); a local #define XML_GE 1 (before the include) unlocks only these two prototypes -- this does not change the behavior of the already-built library and enables no DTD/external/parameter entity parsing: no such handler is registered anywhere in this file (verified by grep), XXE remains unreachable.

Configuration (default + configurable)

Minimum expat version enforced in the build (≥2.4.0) + explicit amplification-protection calls (max factor 100x, activation threshold 8 MiB), not exposed as a user knob (safe values aligned with expat's own defaults, a hardening fix rather than a configurable feature).

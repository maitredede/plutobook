V18 · maxPageCount default beyond the Cairo PDF writer's limit

Medium   Output integrity

Status fixed
Key file include/plutobook.hpp (EngineLimits::maxPageCount)

Nature

Cairo 1.18.4's PDF writer corrupts the xref table / trailer beyond roughly 65536 pages in a single document (a Cairo bug, no crash). The maxPageCount = 100000 default introduced in V09 exceeds this threshold: a document that reaches the cap produces a corrupted (but harmless) PDF.

Location

  • include/plutobook.hpp — EngineLimits::m_maxPageCount = 100000 (V09)
  • subprojects/cairo (1.18.4) — PDF writer: corrupted xref/trailer beyond ~65536 pages

Risk & impact

Impact: output integrity (unreadable PDF). No crash/OOM.

Reproduction

  1. Render to PDF a document exceeding 65536 pages with the default → invalid PDF (pdfinfo/poppler).

Example files:

  • poc/note.md

Proposed fix

Lower the maxPageCount default to a value that guarantees a valid Cairo PDF (≤65536).

Configuration (default + configurable)

maxPageCount default lowered to 65533 (empirically verified by bisection: 65533 pages -> valid PDF, 65534 -> corrupted PDF, reproducible across several document shapes), still configurable via EngineLimits.

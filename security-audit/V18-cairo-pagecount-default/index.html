V18 · Defaut maxPageCount au-dela de la limite du writer PDF Cairo

Moyenne   Integrite de sortie

Statut a corriger
Fichier cle include/plutobook.hpp (EngineLimits::maxPageCount)

Nature

Le writer PDF de Cairo 1.18.4 corrompt la table xref / le trailer au-dela d'environ 65536 pages dans un meme document (bug Cairo, sans crash). Le defaut maxPageCount = 100000 introduit en V09 depasse ce seuil : un document atteignant le plafond produit un PDF corrompu (mais inoffensif).

Localisation

  • include/plutobook.hpp — EngineLimits::m_maxPageCount = 100000 (V09)
  • subprojects/cairo (1.18.4) — writer PDF: xref/trailer corrompu au-dela de ~65536 pages

Risque & impact

Impact : integrite de sortie (PDF illisible). Pas de crash/OOM.

Reproduction

  1. Rendre en PDF un document depassant 65536 pages avec le defaut → PDF invalide (pdfinfo/poppler).

Fichiers d'exemple :

  • poc/note.md

Correctif propose

Abaisser le defaut maxPageCount a une valeur garantissant un PDF Cairo valide (≤65536).

Configuration (defaut + configurable)

Defaut maxPageCount abaisse (ex. 65535), toujours configurable via EngineLimits.
